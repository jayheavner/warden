# Warden v1.1 Hardening Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development or superpowers:executing-plans. Steps use checkbox syntax.

**Goal:** Implement `docs/superpowers/specs/2026-07-16-v11-hardening-design.md`: LaunchDaemon auto-refresh (D), reference-transaction hook delivered via rendered includeIf (H), status/health (S), R3 re-statement — inert until `sudo ./install.sh`.

**Architecture:** A root-owned bash `reference-transaction` hook (fail-open, deny-only-on-positive-match) reaches adopted repos through `/etc/gitconfig → include → warden.gitconfig (rendered includeIf per repo) → hookpath.gitconfig (core.hooksPath)`. A LaunchDaemon (WatchPaths + StartInterval) runs `warden refresh`, which renders settings + registry + gitconfig and drops a health file `warden status` reads.

**Tech Stack:** bash, python3 (macOS CLT, stdlib only), launchd, `python3 -m unittest`.

## Global Constraints

- Spec §3: hook **fails open** on internal error; exits 1 only on positive classification.
- Spec §3: one `git worktree list --porcelain` per transaction, not per ref.
- Spec §3: policy applies only to `refs/heads/*`; detached worktrees never match.
- Spec §3: `/etc/gitconfig` gets exactly one warden include line pair; never touch other content.
- Spec §2: plist PATH pinned to `/usr/bin:/bin:/usr/sbin:/sbin`; log capped 512KiB→64KiB tail.
- v1 constraints carry over: stdlib-only python, no command-text parsing, quote the space in `/Library/Application Support/ClaudeCode` everywhere, audit JSONL schema `ts,session_id,cwd,tool,target,verdict,rule`.
- git floor ≥2.28 for the hook; machine git verified 2.50.1 on 2026-07-16 (empirical, not memory).

**policy:workspace-isolation** (source: project policy, verbatim): "any task that writes code opens its own discrete git worktree — never inherit, reuse, or share another task's worktree — and worktrees are created only inside the project folder (under <project>/.worktrees/, one per task, removed after merge)." Application here: this project's worktree area is `.claude/worktrees/` (warden's own guard recognizes it; `<project>/.worktrees/` would be denied by the frozen zone). Each task: `git worktree add .claude/worktrees/v11-t<N> -b v11-t<N> v11-hardening`, work there, merge back to `v11-hardening`, remove the worktree.

**policy:dependency-freshness** (source: project policy, verbatim): "versions verified current by web search (LTS + security advisories) and pinned exactly; never from memory; no ranges." Application: zero third-party dependencies are added; the only version floor is git ≥2.28, satisfied by the machine's verified 2.50.1.

**Security pass (pre-implementation):** no secrets exist anywhere in this work; hook stdin is treated as data (never eval'd; ref names printed via printf/python, and JSON is emitted by python `json.dumps`, never string-spliced); `/etc/gitconfig` is edited only via `git config --file` (no hand parsing); all installed artifacts root-owned, 0644 (configs) / 0755 (executables); errors shown to sessions name the rule and fix, never internals.

```bash
#!/bin/bash
# Writes chaining dispatchers for the standard git hook names into $1.
# core.hooksPath replaces a repo's own hooks dir wholesale; these shims give
# repo-local hooks their turn back. reference-transaction is NOT written here
# (the policy script owns that name and chains itself).
set -euo pipefail
dest="${1:?usage: make-dispatchers.sh <dest-dir>}"
mkdir -p "$dest"
for h in applypatch-msg pre-applypatch post-applypatch pre-commit \
         prepare-commit-msg commit-msg post-commit pre-rebase post-checkout \
         post-merge pre-push pre-auto-gc post-rewrite fsmonitor-watchman \
         post-index-change; do
  cat > "$dest/$h" <<'DISPATCH'
#!/bin/bash
# warden chaining dispatcher: run the repo's own hook of this name, if any.
common="$(git rev-parse --git-common-dir 2>/dev/null)" || exit 0
h="$common/hooks/$(basename "$0")"
[ -x "$h" ] && exec "$h" "$@"
exit 0
DISPATCH
  chmod 0755 "$dest/$h"
done
```

- [ ] **Step 4: Write `githooks/reference-transaction`:**

```bash
#!/bin/bash
# warden R1 hook (reference-transaction): refuse moving refs/heads/<b> when
# <b> is checked out in a worktree other than the one this command runs in.
# Fail-open: any internal error exits 0 (a broken hook must never become a
# machine-wide git outage); exit 1 only on positive classification.
# Chains to the repo's own hook afterward. Spec: v1.1 design §3.
state="${1:-}"
input="$(cat 2>/dev/null || true)"

deny_reason=""
if [ "$state" = "prepared" ]; then
  wtlist="$(git worktree list --porcelain 2>/dev/null || true)"   # once per transaction
  here="$(git rev-parse --show-toplevel 2>/dev/null || true)"
  if [ -n "$wtlist" ] && [ -n "$here" ]; then
    while IFS= read -r line; do
      [ -n "$line" ] || continue
      ref="${line##* }"
      case "$ref" in refs/heads/*) ;; *) continue ;; esac
      wt="$(printf '%s\n' "$wtlist" | awk -v want="branch $ref" '
        index($0, "worktree ") == 1 { w = substr($0, 10) }
        $0 == want { print w; exit }')"
      [ -n "$wt" ] || continue                                    # detached/unmatched: no opinion
      if [ "$wt" != "$here" ]; then
        deny_reason="warden R1: refusing to move $ref — that branch is checked out at $wt and this command runs at $here. Commit in that worktree instead."
        break
      fi
    done <<INPUT
$input
INPUT
  fi
fi

if [ -n "$deny_reason" ]; then
  printf '%s\n' "$deny_reason" >&2
  audit="${WARDEN_AUDIT_FILE:-$HOME/.claude/warden/audit.jsonl}"
  {
    mkdir -p "$(dirname "$audit")" 2>/dev/null
    python3 - "$audit" "$PWD" "$deny_reason" <<'PY' 2>/dev/null
import datetime, json, sys
rec = {"ts": datetime.datetime.now().astimezone().isoformat(),
       "session_id": "", "cwd": sys.argv[2],
       "tool": "git-reference-transaction", "target": sys.argv[3],
       "verdict": "deny", "rule": "R1"}
open(sys.argv[1], "a").write(json.dumps(rec, ensure_ascii=False) + "\n")
PY
    logger -t warden "R1 deny: cwd=$PWD" 2>/dev/null
  } || true
  exit 1
fi

# Chain to the repo's own reference-transaction hook (common hooks dir — NOT
# core.hooksPath, which points back at us).
common="$(git rev-parse --git-common-dir 2>/dev/null || true)"
self_dir="$(cd "$(dirname "$0")" 2>/dev/null && pwd || true)"
repo_hook="$common/hooks/reference-transaction"
if [ -n "$common" ] && [ -x "$repo_hook" ] && [ "$common/hooks" != "$self_dir" ]; then
  printf '%s\n' "$input" | "$repo_hook" "$state"
  exit $?
fi
exit 0
```

```python
def render_gitconfig(repos, managed_root):
    """One includeIf stanza per adopted repo; trailing / on the gitdir
    pattern matches the repo's .git and every linked worktree's gitdir."""
    hookpath = managed_root + "/warden/hookpath.gitconfig"
    lines = ["# rendered by warden render.py -- do not edit; sudo warden refresh"]
    for r in repos:
        lines += ['[includeIf "gitdir:%s/"]' % r["root"],
                  "\tpath = %s" % hookpath]
    return "\n".join(lines) + "\n"


def _atomic_write_text(path, text):
    tmp = path + ".tmp"
    with open(tmp, "w") as f:
        f.write(text)
    os.replace(tmp, path)
```

```bash
WD="$DEST/warden"
DAEMON_LABEL="com.warden.refresh"

health() {  # health <ok|fail> <detail...>

import datetime, json, sys
path, verdict = sys.argv[1], sys.argv[2]
rec = {"ok": verdict == "ok",
       "ts": datetime.datetime.now().astimezone().isoformat()}
if verdict == "ok":

    ms = json.load(open(sys.argv[4]))

else:

tmp = path + ".tmp"
json.dump(rec, open(tmp, "w"))

import datetime, json, os, sys
def age(iso):
```

```bash
#!/bin/bash
# Sourceable helpers: the ONLY code that touches /etc/gitconfig. Edits via
# `git config --file` exclusively; never touches other content (v1.1 spec §3).

warden_include_add() {  # <etcfile> <includepath>
  local f="$1" inc="$2"
  if ! git config --file "$f" --get-all include.path 2>/dev/null | grep -qxF "$inc"; then
    git config --file "$f" --add include.path "$inc"
  fi
}

warden_include_remove() {  # <etcfile> <includepath>
  local f="$1" inc="$2" esc
  [ -f "$f" ] || return 0
  esc="$(printf '%s' "$inc" | sed 's/[][\.*^$/]/\\&/g')"
  git config --file "$f" --unset-all include.path "^${esc}$" 2>/dev/null || true
  # drop a now-empty [include] section header / whole file
  if ! grep -q '[^[:space:]]' "$f" 2>/dev/null \
     || [ "$(grep -cv '^\[include\]$\|^[[:space:]]*$' "$f")" = 0 ]; then
    rm -f "$f"
  fi
}
```
